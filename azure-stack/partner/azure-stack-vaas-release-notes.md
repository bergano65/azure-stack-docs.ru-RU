# <a name="release-notes-for-validation-as-a-service"></a>Заметки о выпуске для проверки как службы

[!INCLUDE [Azure_Stack_Partner](./includes/azure-stack-partner-appliesto.md)]

Эта статья содержит заметки о выпуске для проверки как службы Azure Stack.

## <a name="version-4421"></a>Версия 4.4.2.1

9 января 2020 г.

- Проверка обновлений содержимого
  - Рабочий процесс проверки OEM (версия 5.1.52.0 -> 5.1.53.0). Уменьшено количество необходимых параметров на панели "расписание тестирования". 
  - Исправление ошибки в тесте вычислений (TestVMOperations)
    
- Известные проблемы
  - Обратитесь по адресу vaashelp@microsoft.com, если во время рабочего процесса проверки OEM не удается выполнить следующие тестовые случаи:
    - Test101LinuxEmptyAttachedDiskManagedDisk;
    - Test101WindowsEmptyAttachedDiskManagedDisk.

3 декабря 2019 г.

- Проверка обновлений содержимого
  - Обновлена интерактивная документация по рабочему процессу ежемесячных обновлений Azure Stack и рабочему процессу проверки пакетов OEM. Ознакомьтесь с обновленной документацией по проверке пакетов OEM и проверке обновлений программного обеспечения от корпорации Майкрософт.
  - Обновление рабочего процесса проверки пакетов VaaS: рабочий процесс проверки OEM является единственным тестом, необходимым для ежемесячной проверки обновлений Azure Stack и проверки пакетов OEM. Тест обновляет метку с помощью предоставленных пакетов Azure Stack и OEM и запускает проверочные тесты механизма имитации в облаке.
  - Обновление расширения VaaS PowerShell: теперь поддерживается автоматизация рабочих процессов проверки пакетов. Для получения дополнительных сведений о расположении и пошаговых инструкций по использованию этого расширения ознакомьтесь с автоматизацией Azure Stack VaaS с помощью Powershell.

- Известные проблемы
  - Обратитесь по адресу vaashelp@microsoft.com, если во время рабочего процесса проверки OEM не удается выполнить следующие тестовые случаи:
    - Test101LinuxEmptyAttachedDiskManagedDisk;
    - Test101WindowsEmptyAttachedDiskManagedDisk.


## <a name="version-4353"></a>Версия 4.3.5.3

7 ноября 2019 г.

- Проверка обновлений содержимого
  - Проверка ежемесячных обновлений Azure Stack (версия 5.1.46.0 -> 5.1.49.0).
  - Проверка пакетов для расширения OEM (версия 5.1.46.0 -> 5.1.49.0).
  - Результаты для версии 5.1.46.0 сохранены. При успешном выполнении в версии 5.1.46.0 сообщите на электронный адрес vaashelp@microsoft.com при отправке результатов.

- Исправления ошибок
  - Исправлена проблема, из-за которой не удавалось выполнить проверку ежемесячных обновлений Azure Stack, если в update.zip содержались специальные символы.

- Известные проблемы
  - Тесты VaaS завершаются ошибкой, если программа mstest.exe не найдена. Обходное решение.
    1. Скопируйте (CTRL+C) агент в окне PowerShell.
    1. Введите mstest.exe, чтобы убедиться, что программа mstest.exe распознана.
    1. Если программа mstest.exe не распознана, закройте текущее окно PowerShell.
    1. Нажмите кнопку "Пуск" (не PowerShell на панели задач), найдите программу PowerShell и откройте ее с правами администратора.
    1. Введите mstest.exe и убедитесь, что она доступна в виде команды.
    1. Перезагрузите агент, а затем перезапустите тест.
  - Иногда механизм имитации в облаке сообщает о сбоях при \*тестировании виртуальной машины. Прежде чем выполнять повторный запуск, обратитесь по этому адресу vaashelp@microsoft.com. 


29 октября 2019 г.

- Обновлена интерактивная документация по рабочему процессу ежемесячных обновлений Azure Stack и рабочему процессу проверки пакетов OEM.

    Ознакомьтесь с обновленной документацией по проверке пакетов OEM и проверке обновлений программного обеспечения от корпорации Майкрософт.
- Обновление рабочего процесса VaaS. Ежемесячное обновление Azure Stack (версия 5.1.30.0 -> 5.1.46.0) — рабочий процесс теста проверки ежемесячных обновлений Azure Stack обновлен.

    Для рабочего процесса больше не требуется ручное вмешательство, и он может быть запланирован для беспрепятственного выполнения.
- Обновление рабочего процесса VaaS. Проверка пакетов OEM (версия 5.1.30.0 -> 5.1.46.0) — рабочий процесс проверки пакета OEM обновлен.

    Для рабочего процесса больше не требуется ручное вмешательство, и он может быть запланирован для беспрепятственного выполнения.
- Механизм имитации в облаке в рабочем процессе проверки пакетов OEM (версия 5.1.30.0 -> 5.1.46.0) был обновлен для ускорения проверки. Время выполнения уменьшено до 1 часа.
- Для механизма имитации в облаке в рабочем процессе проверки пакетов OEM и рабочего процесса обновлений Azure Stack (версия 5.1.30.0 -> 5.1.46.0) нужно, чтобы обновления проверялись в двух разных родительских папках при отсутствии других обновлений в дочерних папках.
- Для механизма имитации в облаке в рабочем процессе проверки пакетов OEM и рабочем процессе обновлений Azure Stack (версия 5.1.30.0 -> 5.1.46.0) нужно, чтобы тесты были запланированы в следующем порядке: тест проверки ежемесячных обновлений Azure Stack, тест проверки пакетов для расширения OEM и, наконец, механизм имитации в облаке.
- Обновление агента VaaS. Обновленный агент VaaS теперь использует учетные данные администратора облака Azure Stack для запроса метки, чтобы получить сведения о ней для автоматического заполнения рабочих процессов. 

    Для этого обновления необходимо обновить и перезапустить все агенты. Инструкции по обновлению агента VaaS см. по этой ссылке https://docs.microsoft.com/azure-stack/partner/azure-stack-vaas-local-agent.
- Обновление пользовательского интерфейса портала VaaS. Таблица выбора агента была перемещена в расположение над панелью планирования тестов для облегчения тестирования.

    При планировании задания больше не требуется вводить сведения о метке, если агенты VaaS были правильно обновлены.

## <a name="version-402"></a>Версия 4.0.2

7 января 2019 г.

Если вы запускаете рабочий процесс проверки ежемесячного обновления Azure Stack, а версия вашего пакета обновлений изготовителя оборудования не 1810 или выше, вы получите сообщение об ошибке, как только перейдете к шагу обновления изготовителя оборудования. Это ошибка. Исправление находится на стадии разработки. Ниже указаны шаги по устранению ошибки:

1. Запустите обновление изготовителя оборудования в обычном режиме.
2. Выполните Test-AzureStack после успешного применения пакета и сохраните выходные данные.
3. Отмените тест.
4. Отправьте сохраненные выходные данные в VaaSHelp@microsoft.com, чтобы получить результаты передачи для запуска.

## <a name="version-401"></a>Версия 4.0.1

8 октября 2018 г.

- Предварительные требования для VaaS

    `Install-VaaSPrerequisites` больше не требуются учетные данные администратора облака. Если вы используете последнюю версию этого командлета, см. раздел [Скачивание и установка локального агента](azure-stack-vaas-local-agent.md#download-and-install-the-local-agent) с исправленными командами для установки необходимых компонентов. Используйте следующие команды:

    ```powershell
    $ServiceAdminCreds = New-Object System.Management.Automation.PSCredential "<aadServiceAdminUser>", (ConvertTo-SecureString "<aadServiceAdminPassword>" -AsPlainText -Force)
    Import-Module .\VaaSPreReqs.psm1 -Force
    Install-VaaSPrerequisites -AadTenantId $AadTenantId `
                              -ServiceAdminCreds $ServiceAdminCreds `
                              -ArmEndpoint https://adminmanagement.$ExternalFqdn `
                              -Region $Region
    ```

## <a name="version-400"></a>Версия 4.0.0

29 августа 2018 г.

- Изменения для предварительных требований VaaS и виртуального жесткого диска

    Теперь `Install-VaaSPrerequisites` требуются учетные данные администратора облака для устранения проблемы при проверке пакета. В документацию из раздела [Скачивание и установка локального агента](azure-stack-vaas-local-agent.md#download-and-install-the-local-agent) были внесены следующие изменения:

    ```powershell
    $ServiceAdminCreds = New-Object System.Management.Automation.PSCredential "<aadServiceAdminUser>", (ConvertTo-SecureString "<aadServiceAdminPassword>" -AsPlainText -Force)
    $CloudAdminCreds = New-Object System.Management.Automation.PSCredential "<cloudAdminDomain\username>", (ConvertTo-SecureString "<cloudAdminPassword>" -AsPlainText -Force)
    Import-Module .\VaaSPreReqs.psm1 -Force
    Install-VaaSPrerequisites -AadTenantId $AadTenantId `
                              -ServiceAdminCreds $ServiceAdminCreds `
                              -ArmEndpoint https://adminmanagement.$ExternalFqdn `
                              -Region $Region `
                              -CloudAdminCredentials $CloudAdminCreds
    ```
    > [!NOTE]
    > Необходимые сценарию `$CloudAdminCreds` предназначены для проверяемого экземпляра Azure Stack. Они отличаются от учетных данных Azure Active Directory, используемых клиентом VaaS.

- Обновление локального агента

    Предыдущая версия локального агента несовместима с текущей версией службы 4.0.0. Все пользователи должны обновить свои локальные агенты. Инструкции по установке последней версии агента см. в разделе [Скачивание и установка локального агента](azure-stack-vaas-local-agent.md#download-and-install-the-local-agent).

- Обновление автоматизации PowerShell

    Были внесены изменения сценарии PowerShell `LaunchVaaSTests`, которым требуется последняя версия пакетов создания сценариев. Инструкции по установке последней версии пакета создания сценариев см. в разделе [Запуск процесса прохождения теста](azure-stack-vaas-automate-with-powershell.md).

- Портал проверки как услуги

  - Уведомления о подписании пакета

    Когда пакет настройки OEM отправляется в рамках рабочего процесса проверки пакета, формат пакета будет проверен, чтобы подтвердить его соответствие опубликованной спецификации. Если пакет не соответствует, происходит сбой выполнения. Уведомления по электронной почте направляются на адрес электронной почты зарегистрированного контактного лица Azure Active Directory для клиента.

  - Категория интерактивных тестов

    Добавлена категория **интерактивных** тестов. Эти тесты позволяют выполнять неавтоматические интерактивные сценарии Azure Stack.

  - Интерактивная проверка функций

    Теперь в рабочем процессе тестового прохода доступна возможность отправки целевых отзывов по определенным функциям. Тест `OEM Update on Azure Stack 1806 RC Validation 5.1.4.0` проверяет, правильно ли применены определенные обновления, и собирает отзывы.
